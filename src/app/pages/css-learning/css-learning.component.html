CSS Grid Layout

CSS Grid is a powerful two-dimensional layout system designed for organizing content in rows and columns. Unlike Flexbox which is primarily one-dimensional, Grid allows precise control over both dimensions simultaneously.

Key Grid Concepts:

  • Grid Container: The element with display: grid that becomes the grid container.
  • Grid Items: Direct children of the grid container that are placed in the grid.
  • Grid Lines: The horizontal and vertical lines that divide the grid and separate the rows and columns.
  • Grid Tracks: The space between two adjacent grid lines (rows or columns).
  • Grid Cell: The intersection of a row and column, the smallest unit of the grid.
  • Grid Area: Any rectangular area bounded by four grid lines, can contain one or more grid cells.

CSS Flexbox Layout

Flexbox is a one-dimensional layout system that helps distribute space and align items within a container, even when their size is unknown or dynamic.

Key Flexbox Concepts:

  • Flex Container: The element with display: flex that contains flex items.
  • Flex Items: The direct children of a flex container.
  • Main Axis: The primary axis along which flex items are laid out (horizontal by default).
  • Cross Axis: The perpendicular axis to the main axis.
  • flex-direction: Establishes the main axis (row, row-reverse, column, column-reverse).
  • justify-content: Aligns items along the main axis.
  • align-items: Aligns items along the cross axis.

Flexbox is particularly useful for:

  • Navigation menus and toolbars
  • Card layouts that need to be responsive
  • Centering content vertically and horizontally
  • Creating flexible form layouts

In the example below, we're using Flexbox to create an evenly spaced row of items that will wrap responsively as the screen size changes.

When to use Grid vs. Flexbox?

  • Use Grid when you need to control layout in two dimensions (rows AND columns).
  • Use Flexbox when you need to arrange items in a single direction (row OR column) with flexible sizing.
  • Use Both together for complex layouts - Grid for the overall page structure, Flexbox for components within grid areas.

CSS Variables (Custom Properties)

CSS Variables (officially called Custom Properties) allow you to store values that can be reused throughout your stylesheet.

Key Benefits:

  • Reduce repetition in your CSS
  • Make your code more maintainable
  • Enable runtime changes with JavaScript
  • Create theme variations easily

Variables are defined with --name and accessed with var(--name). They follow the cascade and can be scoped to elements or declared globally in :root.
