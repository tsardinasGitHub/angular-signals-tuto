Signal Inputs in Angular

Learn how to use Angular's new input() and model() functions with signals for component communication

What are Signal Inputs?

Signal Inputs are a new way to declare component inputs in Angular, introduced in Angular 17. They leverage the reactivity system of signals to provide a more concise and type-safe way to define component inputs compared to the traditional @Input() decorator.

Benefits of Signal Inputs

  • More concise syntax for defining inputs
  • Better type safety with required and optional inputs
  • Built-in transform functions for input values
  • Seamless integration with Angular's signals for reactive programming
  • Two-way binding support with model() signals
